## CandyMonitor 1.0.10

- 新增局域网 IonBridge 发现与直连读取：优先通过 `cp02-<psn>.local` 或局域网扫描读取小电拼本地接口，失败时继续回落 MCP。
- 优化首页功率拓扑：能量流线条改为同色、仅用线宽区分强弱，idle 虚线在深色模式下更清晰，同时修复 CP02/Ultra 图示自适应映射。
- 修复多端口记录展示：首页和菜单不再只显示第一条“正在记录”，新会话创建后会立即选中并清空旧图表缓存，避免旧记录把新记录顶回去。
- 优化充电记录图表性能：详情页直接渲染预降采样后的完整曲线，减少点进记录后等待数秒或先显示空白的问题。
- 首页实时采样曲线只保留最近 10 秒，充电记录仍保留完整会话曲线；实时曲线移除面积填充，只保留线图。
- 标题徽章在有功率输出时随功率旋转，低功率慢、高功率快，idle 自动停止，并微调与标题字体的垂直对齐。
- 发布前重新验证 App Sandbox entitlement，确保设备列表和历史记录继续读取 sandbox 容器数据。

## CandyMonitor 1.0.9

- 优化图表悬停（Hover）体验：解决在图表更新、曲线向左平移时悬停虚线及显示数值不同步刷新问题，实现随时间平移自动动态更新。
- 优化首页实时监控图表：将数据时间展示窗口由 30 分钟缩短为最近 1 分钟（60秒），以便更好地展现瞬态硬件波形物理参数。
- 性能优化重构：使用增量 EMA 算法平滑计算，并将图表渲染组件与指示虚线进行隔离与防重绘优化，大幅降低高频滚动与悬停时的 CPU 占用。
- 修复 `ChartLinePoint` 属性不满足 `Equatable` 导致在 Release 构建时可能出现的模板类实例化错误。
- 重新验证了 Sandbox 安全沙盒特性，确保所有配置与充电历史数据无损加载。

## CandyMonitor 1.0.4

- 修复同一设备同一端口可能出现多条“正在记录”的问题：记录前会从 SwiftData 查找该端口已有未结束会话，不再只依赖内存状态。
- 启动和轮询时会自动清理历史重复未结束会话：保留最早的有效记录，迁移重复记录下的采样点，重算峰值/平均值/协议/电量统计后删除重复行。
- AGENTS 和 README 补充充电会话唯一性规则，要求后续改动不能只在 UI 层隐藏重复记录，也不能删除历史采样曲线。

## CandyMonitor 1.0.3

- 接入小程序同源 IOT WebSocket PD status：监听 service `130` / `stream_port_pd_status`，用于补充 Apple/PD 设备的电池、容量、健康度和预计时间等字段。
- 新增可选“小程序 IOT WS JWT”本地加密配置；有 WS JWT 时优先使用同源 WS 数据，没有或连接失败时继续回落到 MCP `get_port_pd_status`。
- WS PD 字段按小程序语义归一化端口号与字段名，并和 MCP PD 数据合并，避免单一来源缺字段时详情页空白。
- AGENTS、README 和本地 `.agent` 规则补充 WS/JWT 与持久化注意事项，明确 MCP SSE 短 token 不是 IOT WS JWT。

## CandyMonitor 1.0.2

- 修复 release 二次签名丢失 App Sandbox entitlement 导致更新后读取非沙盒空库的问题；本地设备、充电记录和采样数据会继续从原 sandbox container 读取。
- `scripts/build.sh` 现在使用 `CandyMonitor/CandyMonitor/CandyMonitor.entitlements` 重新签名，并在打包前强制验证 `com.apple.security.app-sandbox=true`，缺失时直接终止发布。
- 端口详情的标准 PD 电池字段对齐小程序命名，补充读取 `电池设计容量`、`当前最大容量`、`当前电量`、`健康度` 和 `预计充满时间` 等字段。
- 快充协议名称改为小程序同款枚举显示，例如 `PD_FIXHV`、`PD_PPS`、`NOT_CHARGING`；小米私有协议仍显示为 `小米澎湃秒充`。
- AGENTS、README 和本地 `.agent` 发布文档补充持久化数据域与 entitlement 发布检查，防止后续 release 再把数据目录切错。

## CandyMonitor 1.0.1

- 充电记录只在端口实际输出功率后创建，避免单口接入时多个空闲端口一起显示“正在记录”。
- 充电记录列表改为桌面式选择交互：点击单选，按住 Command 增减多选，按住 Shift 连续选择。
- 首页实时曲线改为仅展示过去半小时，绘制前按端口保留关键采样点，降低长时间运行后的 hover 和重绘卡顿。
- 实时曲线 hover 改为显示同一时间附近所有可见端口读数，不再总是只落到 A 口。
- 端口详情扩展标准 PD 设备/电池字段解码，并在详情弹窗补充读取端口统计，Apple 设备型号、品牌、电量、容量、健康度和预计充满等字段会尽可能显示。
- 性能热路径重做：机器静态信息改为缓存刷新，轮询采样不再每秒全量重读会话，SwiftData 保存改为按会话边界和短周期合并，会话详情曲线也做关键点降采样。

## CandyMonitor 1.0

- Hotfix build 5：修复旧库迁移失败导致设备列表变空、手动添加无反应的问题。
- 新增采样字段改为向后兼容的可迁移结构，旧版 `PortSample` 记录不再卡住 SwiftData store 加载。
- 启动、窗口出现和回到前台都会重新加载本地设备；同时保存一份不含 MCP 地址的设备元数据索引，用于数据库异常时自动补回设备行。
- 本地加密 MCP vault 支持从旧 Application Support 位置迁入 sandbox 容器，减少安装方式切换后的重新配置概率。
- Hotfix build 4：继续收敛实时轮询中 SwiftData fault 触发的崩溃，并减少空闲端口每秒写库。
- 实时保存改为先用普通 Swift 值更新会话统计，再插入采样对象，避免刚插入或已 fault 的 SwiftData 对象在同一轮保存里互相读写。
- 端口详情中移除没有动作的“设备详情 / 专业信息 / 协议管理”假入口。
- 功率拓扑端口卡在接入设备时显示该端口实时瓦数，充电记录详情的“样本”卡改为“实时功率”。
- 实时采样曲线增加空载死区和最小功率量程，避免 0.03W 级噪声把曲线放大成跳动。
- 实时采样曲线按端口、接入状态和采样时间断点分段，并改为线性连接，避免跨空白区拉出夸张弯线。
- 端口接入状态只信 MCP 明确上报的 `connected` / 充电位，不再用 0.05W 低功率噪声推断 active。
- 充电记录列表支持多选删除。
- MCP 未上报 `enable` 时，端口详情和控制台不再显示推测性的端口开关按钮。
- MCP SSE 地址改为本地 AES-GCM 加密文件保存，不再使用 macOS Keychain，避免新版安装后反复弹钥匙串授权。
- 充电记录页改为不透明原生分栏，并支持给单条充电记录重命名。
- 充电记录改为端口接入后立即创建，拔掉后按端口接入状态结束，不再被 0.5W 功率阈值卡住。
- 充电记录详情支持删除整条记录及其采样点。
- 轮询写入会话时不再长期持有 SwiftData 可变对象，降低记录刷新期间的 CoreData fault 崩溃风险。
- 0 条充电记录时使用统一空态，不再显示割裂的左右色块。
- 设置里的显示名称支持回车和失焦保存，无需再点“保存并校验”。
- 端口开关状态未知但已接入设备时，详情页显示“已接入”并只保留关闭入口，不再误导为需要开启。
- 首个 macOS 原生版本：提供小电拼设备监控、端口详情、充电记录和控制台。
- 复刻并优化 CoCan Mirror / 小电拼 4C1A 功率拓扑视图，使用真实产品素材。
- 新增本地 SwiftData 充电会话记录和 CSV 导出。
- 提供端口开关、温控模式、充电策略、功率分配和线补控制入口。
- 修正端口开关语义：不再把“是否正在充电”误认为“端口是否开启”。
- 提供独立 `arm64` 和 `x86_64` DMG 安装包。
