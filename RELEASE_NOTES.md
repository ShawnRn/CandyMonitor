## CandyMonitor 1.0

- Hotfix build 5：修复旧库迁移失败导致设备列表变空、手动添加无反应的问题。
- 新增采样字段改为向后兼容的可迁移结构，旧版 `PortSample` 记录不再卡住 SwiftData store 加载。
- 启动、窗口出现和回到前台都会重新加载本地设备；同时保存一份不含 MCP 地址的设备元数据索引，用于数据库异常时自动补回设备行。
- 本地加密 MCP vault 支持从旧 Application Support 位置迁入 sandbox 容器，减少安装方式切换后的重新配置概率。
- Hotfix build 4：继续收敛实时轮询中 SwiftData fault 触发的崩溃，并减少空闲端口每秒写库。
- 实时保存改为先用普通 Swift 值更新会话统计，再插入采样对象，避免刚插入或已 fault 的 SwiftData 对象在同一轮保存里互相读写。
- 端口详情中移除没有动作的“设备详情 / 专业信息 / 协议管理”假入口。
- 功率拓扑端口卡在接入设备时显示该端口实时瓦数，充电记录详情的“样本”卡改为“实时功率”。
- 实时采样曲线增加空载死区和最小功率量程，避免 0.03W 级噪声把曲线放大成跳动。
- 实时采样曲线按端口、接入状态和采样时间断点分段，并改为线性连接，避免跨空白区拉出夸张弯线。
- 端口接入状态只信 MCP 明确上报的 `connected` / 充电位，不再用 0.05W 低功率噪声推断 active。
- 充电记录列表支持多选删除。
- MCP 未上报 `enable` 时，端口详情和控制台不再显示推测性的端口开关按钮。
- MCP SSE 地址改为本地 AES-GCM 加密文件保存，不再使用 macOS Keychain，避免新版安装后反复弹钥匙串授权。
- 充电记录页改为不透明原生分栏，并支持给单条充电记录重命名。
- 充电记录改为端口接入后立即创建，拔掉后按端口接入状态结束，不再被 0.5W 功率阈值卡住。
- 充电记录详情支持删除整条记录及其采样点。
- 轮询写入会话时不再长期持有 SwiftData 可变对象，降低记录刷新期间的 CoreData fault 崩溃风险。
- 0 条充电记录时使用统一空态，不再显示割裂的左右色块。
- 设置里的显示名称支持回车和失焦保存，无需再点“保存并校验”。
- 端口开关状态未知但已接入设备时，详情页显示“已接入”并只保留关闭入口，不再误导为需要开启。
- 首个 macOS 原生版本：提供小电拼设备监控、端口详情、充电记录和控制台。
- 复刻并优化 CoCan Mirror / 小电拼 4C1A 功率拓扑视图，使用真实产品素材。
- 新增本地 SwiftData 充电会话记录和 CSV 导出。
- 提供端口开关、温控模式、充电策略、功率分配和线补控制入口。
- 修正端口开关语义：不再把“是否正在充电”误认为“端口是否开启”。
- 提供独立 `arm64` 和 `x86_64` DMG 安装包。
